# Examples Directory

This directory contains practical examples for learning `amazing-async-dev`.

---

## Available Examples

| Example | Mode | Purpose | Audience |
|---------|------|---------|----------|
| [single-feature-day-loop](single-feature-day-loop/) | **Direct** | Default onboarding - Complete day loop walkthrough | First-time users |
| [verify-initialization](verify-initialization/) | Starter-Pack | Quick verification of starter-pack initialization | Integration testing |
| [core-objects](core-objects/) | Direct | Schema examples for each artifact type | Developers customizing artifacts |
| [pilot](pilot/) | Starter-Pack | Advisor integration pilot artifacts | Integration testing |

**Modes**: 
- **Direct** = Uses async-dev standalone (no external dependencies)
- **Starter-Pack** = Uses starter-pack file (optional enhancement)

---

## Recommended First Example

**Start with `single-feature-day-loop`** - it demonstrates the complete workflow in one pass:

```
plan-day → run-day → review-night → resume-next-day
```

You'll see:
- ExecutionPack generation
- ExecutionResult output
- DailyReviewPack creation
- RunState updates

**Time to complete**: 5-10 minutes with mock mode.

## Quick Access

```bash
# Run the onboarding example immediately
cd examples/single-feature-day-loop
python demo-day-loop.py --verbose
```

Or run CLI commands manually:

```bash
# From repository root
cd amazing-async-dev

# Initialize (if not done)
python cli/asyncdev.py init create

# Create product
python cli/asyncdev.py new-product create --product-id demo-001 --name "Demo"

# Add feature
python cli/asyncdev.py new-feature create --product-id demo-001 --feature-id hello-world --name "Hello World"

# Plan task
python cli/asyncdev.py plan-day create --product-id demo-001 --feature-id hello-world --task "Create hello-world.txt"

# Run (mock mode for testing)
python cli/asyncdev.py run-day --product-id demo-001 --mode mock

# Review
python cli/asyncdev.py review-night generate --product-id demo-001

# Resume
python cli/asyncdev.py resume-next-day continue-loop --product-id demo-001 --decision approve
```

## Troubleshooting

### "Product already exists"

```
Product already exists: projects/demo-001
```

**Solution**: Use a different product ID or delete the existing directory:

```bash
rm -rf projects/demo-001
```

### "Feature not found"

```
Feature not found: hello-world
```

**Solution**: Create the feature first:

```bash
python cli/asyncdev.py new-feature create --product-id demo-001 --feature-id hello-world --name "Hello World"
```

### "RunState not in executing phase"

```
Cannot run-day: current_phase is planning
```

**Solution**: Plan a task first:

```bash
python cli/asyncdev.py plan-day create --product-id demo-001 --feature-id hello-world --task "..."
```

### "No execution packs found"

```
No execution packs to run
```

**Solution**: Run plan-day before run-day:

```bash
python cli/asyncdev.py plan-day create --product-id demo-001 --feature-id hello-world --task "Create hello.txt"
python cli/asyncdev.py run-day --product-id demo-001 --mode mock
```

### Mock mode shows "status: failed"

This is expected for some mock scenarios. Check `execution-results/*.md` for details.
